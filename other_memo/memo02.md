1. クローンから始まる一連のGitHub作業フロー
2. ブランチを切る
3. push
4. プルリクエスト
5. マージ
6. 補足：mainブランチで編集してしまった場合の対処法

初心者向けに丁寧に説明する必要があります。コマンドと説明を含めて、ステップバイステップで説明します。

GitHub初心者の方向けに、基本的な作業フローを詳しく説明しますね。

## 📚 GitHub作業の基本的な流れ

### 1. リポジトリをクローンする

まず、作業したいリポジトリを自分のPCにコピー（クローン）します。

```bash
# GitHubのリポジトリページで「Code」ボタンをクリックしてURLをコピー
git clone https://github.com/ユーザー名/リポジトリ名.git

# クローンしたフォルダに移動
cd リポジトリ名
```

### 2. 新しいブランチを作成する

**なぜブランチを切るの？**
- mainブランチは本番環境のコードなので、直接編集すると危険
- 自分の作業用の「部屋」を作るイメージです

```bash
# 現在のブランチを確認（最初はmainにいるはず）
git branch

# 新しいブランチを作成して移動（2つの方法）
# 方法1: 作成と移動を同時に行う（推奨）
git checkout -b feature/新機能名

# 方法2: 作成してから移動
git branch feature/新機能名
git checkout feature/新機能名
```

**ブランチ名の付け方の例：**
- `feature/add-login` （機能追加）
- `fix/button-bug` （バグ修正）
- `update/readme` （ドキュメント更新）

### 3. コードを編集する

お好きなエディタでファイルを編集します。

### 4. 変更をコミットする

```bash
# 変更されたファイルを確認
git status

# 変更をステージング（コミット準備）
git add .  # すべての変更を追加
# または特定のファイルだけ追加
git add ファイル名

# コミット（変更を記録）
git commit -m "ログイン機能を追加"
```

### 5. GitHubにプッシュする

```bash
# 初めてこのブランチをプッシュする場合
git push -u origin feature/新機能名

# 2回目以降は
git push
```

### 6. プルリクエストを作成する

1. GitHubのリポジトリページを開く
2. 「Compare & pull request」ボタンが表示されているのでクリック
3. プルリクエストの内容を記入：
   - **タイトル**: 何を変更したか簡潔に
   - **説明**: なぜ変更したか、どう変更したか詳しく
4. 「Create pull request」をクリック

### 7. レビューとマージ

**チーム開発の場合：**
- 他のメンバーにレビューしてもらう
- 承認されたら「Merge pull request」をクリック
- 「Confirm merge」をクリック

**個人開発の場合：**
- 自分で確認後、マージできます

### 8. ローカルのmainブランチを更新

マージ後は、ローカルのmainブランチも最新にしておきましょう。

```bash
# mainブランチに移動
git checkout main

# 最新の状態を取得
git pull origin main

# 不要になったブランチを削除（任意）
git branch -d feature/新機能名
```

## 🆘 mainブランチで編集してしまった場合の対処法

### 方法1: まだコミットしていない場合

```bash
# 現在の変更を一時保存
git stash

# 新しいブランチを作成して移動
git checkout -b feature/新機能名

# 保存した変更を適用
git stash pop

# この後は通常通りコミット、プッシュ
git add .
git commit -m "変更内容"
git push -u origin feature/新機能名
```

### 方法2: すでにコミットしてしまった場合

```bash
# コミットの履歴を確認
git log --oneline

# 新しいブランチを作成（現在のコミットも含まれる）
git checkout -b feature/新機能名

# mainブランチに戻る
git checkout main

# mainブランチを1つ前のコミットに戻す
git reset --hard HEAD~1

# 新しいブランチに戻って作業継続
git checkout feature/新機能名
git push -u origin feature/新機能名
```

## 💡 覚えておくと便利なTips

**よく使うコマンド:**
```bash
# 現在の状態を確認
git status

# ブランチ一覧を見る
git branch -a

# コミット履歴を見る
git log --oneline --graph

# 最新の変更を取得（マージはしない）
git fetch
```

**プルリクエスト作成時のポイント:**
- 変更内容が分かりやすいタイトルを付ける
- なぜこの変更が必要かを説明に書く
- スクリーンショットがあると分かりやすい
- 大きな変更は小さく分けて複数のPRにする

**困ったときは:**
- `git status` で現在の状態を確認
- 間違えても `git reset` や `git revert` で戻せます
- 分からないことは遠慮なく質問しましょう！

これで基本的な流れは完了です。最初は戸惑うかもしれませんが、何回か繰り返すうちに自然とできるようになりますよ。頑張ってください！
